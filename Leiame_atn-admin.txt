--- CONFS
Há uma pasta "confs" onde se encontram arquivos modelos já devidamente configurados para a maioria das situações, renomeie o arquivo index.php1 para index.php (este arquivo que nega acesso deve ser usado na raiz html caso não haja página padrão).
Se optar por substituir o sudoers use "chmod 0444 /etc/sudoers" e "chown root:root /etc/sudoers".
Dar permissão rw rw rw 666 (chmod 666 /etc/dhcpd.conf)para o arquivo /etc/dhcpd.conf.

--- SCRIPTS
dar permissao a+x para 
squid/script/limpa-squid, reinicia_servico, service, squida e squidf
manut/script/limpaiptables, limpa-log.sh e limpa-logback.sh
status_services/servicos
criar link para /usr/bin/ destes arquivos

--- CONFIGURACOES
Edite todos os arquivos config.inc.php setando os parmetros de acordo com o sistema.
use chown -R apache atn-admin e chmod 777 -R atn-admin

--- SUDOERS
Comente a linha que desabilita o requiretty: 
# Defaults requiretty

Adicione as seguintes linhas, verifique com que usuário o Apache está rodando: 
# ps -aux | grep httpd 

Opção 01
##ATN-ADMIN
apache	ALL=(ALL)	NOPASSWD: ALL

OPção 02
# sintaxe padrão
# usuario computador = NOPASSWD: comando
# apache ALL = NOPASSWD: /bin/halt
# Indico que o usuario apache executa o comando /bin/halt sem senha "NOPASSWD" de qualquer
computador "ALL"

#Para efetuar o desligamento/reiniciar
apache ALL=(ALL) NOPASSWD: /sbin/halt
apache ALL=(ALL) NOPASSWD: /sbin/reboot

#Para fazer a leitura dos logs -- Esta parte não é importante, só se você deseja fazer a leitura dos logs, certifique-se do caminho, e se você configurou a saída dos logs do iptables para um arquivo separado.
apache ALL=(ALL) NOPASSWD: /var/log/messages
apache ALL=(ALL) NOPASSWD: /var/log/shorewall-init.log
